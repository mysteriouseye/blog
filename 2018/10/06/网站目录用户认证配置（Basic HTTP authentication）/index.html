网站目录用户认证配置（Basic HTTP authentication）
计算机 引用文 运维安全
计算机 2018-10-06 Read

1. 为什么需要网站目录验证

  • 提高网站安全性
  • 隐藏你不想给别人展示的页面，只有你自己才能进去

2. 安装必备组件

首先 你需要htpasswd,这个组件是apache里面的，所以你需要安装httpd
Centos ：

yum install httpd

Ubuntu :

apt-get install apache2

3. 生成密码文件

随便找个服务器上的目录，最好别是你的网站根目录

cd /tmp
htpasswd -c mypassword.db user#user是你自己取，登录验证时就是这个用户名
#弹出提示：
New password: *******  #输入认证密码
Re-type new password: ********  #再次输入认证密码
chmod 400 mypassword.db #修改认证文件权限
chown nginx:nginx mypassword.db #修改文件的所有者和组 如果你的服务器权限用户名是其它的，那请把nginx:nginx改掉

4. Nginx配置

    server {
           listen       80;  //监听端口为80
           server_name  www.xxx.com;  //虚拟主机网址    
           location / {
                 root   sites/www1;  //虚拟主机网站根目录    
                 index  index.html index.htm;  //虚拟主机首页 
                 auth_basic "secret";  //虚拟主机认证命名   
                 auth_basic_user_file /tmp/mypasswd.db; //虚拟主机用户名密码认证数据库         
            }   
    }

当然上面只是最简单的配置法

如果要指定某个目录配置，比如网站PHP博客后台管理目录，那么你可以像我这样配置

    location ~ .*\.php(\/.*)*$ {
              fastcgi_split_path_info ^(.+\.php)(/.+)$;
              fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
              fastcgi_pass 127.0.0.1:9000;
              fastcgi_index index.php;
              include fastcgi_params;
             }
             location ^~ /admin/ {     //请求网站后台目录     
               auth_basic "Are you sure?";
               auth_basic_user_file /opt/nginxpwd.db;
              location ~ .*\.php(\/.*)*$ {
                fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
                fastcgi_pass 127.0.0.1:9000;
                include fastcgi_params;
                fastcgi_index login.php; //后台目录主页面
              }
        }

这样保证了博客前台可以让他人看到，后台别人进不去

其它配置方法可以自己探索，我个人的配置还是很蠢的，针对typecho

网上的wordpress方法并不适用于我哪个typecho后台管理

5. apache配置

1.修改 httpd.conf 中的虚拟主机配置：添加

    <Directory "/var/www/web/test">
       Options Indexes FollowSymLinks
       AllowOverride AuthConfig
       Order allow,deny
       Allow from all
    </Directory>

2.到 /var/www/web/test 目录下增加.htaccess文件：

    AuthName "password,please"
    AuthType Basic
    AuthUserFile /tmp/mypassword.db //密码文件
    require valid-user //允许文件中所有用户都能访问，如果只要求特定用的用户的话就这样写 Require user 用户名

其它自行探索，我个人不怎么喜欢apache，但看这样子配置并不难
